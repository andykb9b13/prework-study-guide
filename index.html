HTML Notes

  • Meta data exists in the head. This is info the browser sees but the user does not.
  • The visible information that is displayed to the user is in the body
  • Elements are often nested inside each other.
  • Use SEMANTIC Markupwhenever possible. This makes it clearer for screen readers and other programmers.

CSS Notes

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

GIT Notes

  • git status: check which branch you are on
  • git checkout main: move to the main branch
  • git checkout feature/" ": move to the feature/" " branch
  • git add -A: add all of the changes
  • git commit -m "enter message here": create a message to accompany commits
  • git pull origin main: pull the most current code from the hub to whatever branch you are on
  • git push origin feature/" ": push the code from the feature/" " branch to the hub to be reviewed
  • git checkout -b feature/" ": create a new branch called feature/" " and move to it

Some general things to keep in mind

  • Always a good idea to check which branch you are on
  • Make sure you are in the directory you want to code from
  • Make sure you add a commit and message to keep track of what work has been done

JAVASCRIPT Notes

  • The pseudocode example is helpful
    • var hasFuel = false; if (hasFuel == false) { //vehicle will fill up their gas tank console.log("let's get of this exit and get fuel") hasFuel == true }
    • Variables are changeable if they are not const
    • the loop checks the conditon (is hasFuel false?)
    • if the condition is met, it runs the loop
    • once the action has taken place, the variable changes to 'true'
    • it will go up to the top and check again (is hasFuel false?)
    • since the variable changed at the end of the loop, the condition is not met so the loop doesn't run again
  • CONTROL FLOW is the order in which the computer executes code in a script. Code is read from top to bottom.
  • IF statements: code following if statement will only run if the condition is met.
  • ELSE IF and ELSE: Additional conditions with different code following an IF statement.
  • The computer looks to see which condition is truthy and then executes that code.
  • ARRAY: a group of related data.
  • Strings need quotes around them but BOOLEAN values and numbers do not when in an array.
  • INDEX: each item in an array has a position marked by an index value [0, 1, 2, etc]
  • FOR loops: These allow us to repeat an action/iteration based on a set number of times
    1. The first statment is the starting point for the loop. EX. we want to start at the beginning of an array so we will set x=0.
    2. The second statment is the condition. Ex. x is less than the length of the array
    3. The third and final statement is the action to iterate. Ex. x++ (this is shorthand for x+1). Now everytime the code loops, x will increase by 1
  • FUNCTION: groups a set of instructions that tells the computer what to DO. They define and reuse blocks of code.
    • Ex. Function makeTea() { // function body }
    • You then need to call the function. Ex. makeTea() ;
    • Functions can wrap loops and if statments
