Prework Study Guide

HTML Notes

  • Meta data exists in the head. This is info the browser sees but the user does not.
  • The visible information that is displayed to the user is in the body
  • Elements are often nested inside each other.
  • Use SEMANTIC Markupwhenever possible. This makes it clearer for screen readers and other programmers.

CSS Notes

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

GIT Notes

  • git status: check which branch you are on
  • git checkout main: move to the main branch
  • git checkout feature/" ": move to the feature/" " branch
  • git add -A: add all of the changes
  • git commit -m "enter message here": create a message to accompany commits
  • git pull origin main: pull the most current code from the hub to whatever branch you are on
  • git push origin feature/" ": push the code from the feature/" " branch to the hub to be reviewed
  • git checkout -b feature/" ": create a new branch called feature/" " and move to it

Some general things to keep in mind

  • Always a good idea to check which branch you are on
  • Make sure you are in the directory you want to code from
  • Make sure you add a commit and message to keep track of what work has been done
